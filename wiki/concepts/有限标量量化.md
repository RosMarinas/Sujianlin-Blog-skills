# 有限标量量化

## 核心定义

**有限标量量化**（Finite Scalar Quantization, FSQ）是一种为了简化传统自编码器中向量量化（Vector Quantization, VQ）而设计的可微量化算子。不同于 VQ-VAE 使用 $K$ 个高维向量通过最近邻搜索（$\text{argmin}$ 近邻匹配）维护显式码本，FSQ 将离散量化解构为对低维连续隐向量的每个分量独立地执行受限 Sigmoid 缩放和 Round（四舍五入）取整操作，从而隐式构造出包含 $L_1 L_2\cdots L_d$ 个离散网格状态的分类空间。

## 机制对比

| 特性维度 | 向量量化 (VQ) | 有限标量量化 (FSQ) |
|---|---|---|
| **核心算法** | $\mathop{\text{argmin}}$ 码本检索 | $\text{Round}$ 四舍五入标量网格映射 |
| **隐空间维度 $d$** | 通常为高维（如三位数，如 256） | 必须为低维（如个位数，由 $d = \log_L K$ 限制） |
| **显式码本** | 需要维护并优化 $K \times d$ 大小的 Table | 无需显式 Codebook |
| **辅助损失函数** | 必须依靠辅助 Loss 维持隐向量与码本接近 | 无辅助 Loss |
| **梯度估计方式** | 基于最近邻的 STE 替换，极易引发“赢者通吃”坍缩 | 基于 Round 逼近值的 STE 替换，天然免疫坍缩 |

## 适用场景与局限

1. **拟合代偿机制**：
   - 从理论表示能力看，FSQ 强制对编码维度施加了大幅度的降维约束（例如将 $256$ 维投影到 $d=4$ 或 $5$），这会限制特征的纯几何表达力。
   - 然而，在编码器和解码器足够强大（如当前的大参数量 ViT/Transformer 架构）时，网络的深层非线性拟合可以代偿 FSQ 本身的几何劣势。
2. **编码表坍缩的免疫**：
   - 传统 K-Means 聚类或 VQ 匹配在使用大编码表时，由于不可导的 $\text{argmin}$ 导致梯度更新具有硬分配缺陷，大部分码本向量无法获得梯度，形成“赢者通吃”并导致码本坍缩。
   - FSQ 在坐标轴维度直接进行独立的分段等级指派，不涉及样本间的竞争，能够充分利用所有离散编码空间。
3. **效果分水岭**：
   - 在目标编码总量 $K < 1000$ 时，隐空间维度限制成为瓶颈，VQ 表现通常优于 FSQ。
   - 当 $K > 1000$ 且具备复杂的 AE 解码器时，FSQ 的收敛速度、优化稳定性和最终图像 Tokenizer 重构表现全面占优。
