# 通用爬虫探索（一）：适用一般网站的爬虫

## 内容概要
文章探讨了一种摆脱传统正则表达式局限的无监督通用信息抽取方案。核心思想是利用同一网站下不同页面的 DOM 树求交集来构造网站的“标准模板”（固定骨架与公共广告/导航等），并通过页面与模板的差集来提取网页的有效文本。

## 关键内容
1. **爬虫两步走**：1. 下载网页（难点在于应对反爬虫封锁）；2. 信息抽取（难点在于传统正则匹配难以通用）。本系列重点解决第二步。
2. **“做减法”核心原理**：网页内容 - 网页模板 = 网页有效内容。
3. **标准模板交集构造**：
   将页面解析为 DOM 树后，将有文本的节点表示为“标签路径+标签文本”形式（如 `html_head_title_标题文本`），过滤特定样式 id 和 class 的干扰。
   网站标准模板定义为所有网页的交集：
   $$S = \bigcap_{h \in H} h$$
   网页有效文本定义为页面与模板的差集：
   $$\text{effective} = h \setminus S$$
4. **流水式标准模板更新**：无需一次性下载所有网页，初始可用两个页面求交集作为模板，之后每爬取一个新网页即与模板求交更新，使其不断逼近真实骨架。
5. **规则过滤（非自然语言清理）**：
   - *工程规则*：根据中英文字符占比过滤代码。保留含有日期格式的文本；保留纯中文、英文、数字和下划线无空格的用户名文本；保留剩下的部分中中文占比超过阈值的文本。
   - *学术方案*：引入基于二元共现的统计语言模型计算文本概率 $P(w_1, \dots, w_n) = P(w_1) \prod_{i=2}^n P(w_i|w_{i-1})$，设定阈值过滤非自然语言。该方案具有跨语种的数学通用性。
