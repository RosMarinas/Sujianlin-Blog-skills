## 适用问题

在大规模语言模型的混合专家架构（MoE）中，处理靠近输入嵌入层的前几层时。此时向量缺乏长序列上下文信息，容易被极高频 Token 的分配选择主导，导致严重的专家负载不均衡，传统基于词表 ID 的查表映射分配（tid2eid）难以直接应对这种极端偏置。

## 核心变换

将“单变量极其不平衡分布的局部匹配”任务，变换为“高维多元组（组合爆炸）在大空间内的均匀哈希投影”任务。通过提高组合自由度以平滑单个高频符号带来的尖峰。

## 典型步骤

1. 放弃仅根据当前单个 Token Id 进行独立决策的路由方式。
2. 将当前的 Token Id 与其前驱的一个或多个 Token Id 结合，构建 2-Gram $(a,b)$ 或 3-Gram 的组合上下文输入。
3. 选择一个大于总 Token 数量 $m$ 的质数 $q$，使用简单的多项式哈希映射：通过 $(a \cdot q + b) \pmod n$ 得出该上下文组合要分配给的第一个专家（共 $n$ 个专家）。
4. 对同一个输入组合更换不同的质数（或乘系数），重复上述计算 $k$ 次，得到待激活的 $k$ 个多专家列表。

## 直觉

想象你是一所学校的食堂打饭阿姨。如果仅仅按“每个人的姓氏”来分配打饭窗口（单 Token 分配），那一旦遇到全国张姓和李姓占大多数的情况，负责“张”和“李”窗口的阿姨就会忙到飞起，其他窗口却闲得发慌，无论怎么重新分配姓氏组合都无法彻底平衡。那该怎么办？我们可以给规则升维，变成看“姓”加“名”（甚至再加上出生地），即 2-Gram 或 3-Gram。由于姓名的组合数量巨大且分布分散得多，只要我们用一个稍微均匀点的数学公式一顿打散重组，原本聚集在一坨的人群自然就均匀散布到了各个窗口前。

## 边界

- 增加了计算逻辑复杂度。由于路由决策现在依赖上下文，计算不能对单一词典预缓存，需要随序列动态进行。
- 同一输入哈希过程中可能由于模数周期产生重复激活相同专家，这需要微小的碰撞处理或直接忽略。
- 仅在前几层（浅层网络）具有性价比，深层网络已有丰富的上下文特征表示，直接依赖语义向量进行门控通常已经足够分散且对后续逻辑更优。

## 例子

**DeepSeek V4 浅层哈希路由**：
假设总专家 $n=128$，每次激活 $k=4$，如果词汇表中“的”出现频率畸高（占比5%）。如果单独把“的”死死绑定给专家 A，那专家 A 会过载崩溃。利用 2-Gram，我们将“美丽-的”、“伟大-的”、“快速-的”通过大质数哈希 $(id_1 \times q + id_2) \pmod n$，将其彻底打散分配。由于 2-Gram 种类极多，且哈希运算具有强扩散性，最终每个专家分到的 2-Gram 组合频率总和极其接近平均预期 $\frac{k}{n}$，天然实现了完美负载均衡且完全不需记录一个庞大的分配字典表。

## 证据

- ev::11750::极端情况：$p_1 \gg k/n$，即某个Token的频率已经远超均衡水平，此时无论tid2eid怎么编排，都不可能实现负载均衡... 答案很简单，换一种依赖于更多输入信息的Hash方法... 我们可以考虑跟它前面的Token一起，组成“2-gram to expert”。
- ev::11750::以2-gram $(a, b)$为例，一种朴素的Hash方式是：选大于$m$的质数$q$，计算$(aq+b)\mod n$作为激活的Expert，用不同的质数重复$k$次就得到$k$个Expert。
- ev::11750::由于增加了gram数，输入的组合数大大增加，然后映射到有限的$n$个数中，每个数大概率都能被“塞满”，所以只要Hash函数不是特别糟糕，那么结果几乎就是均衡的。因此，它的优点是不用考虑频率，也不用事先记忆一个tid2eid。
