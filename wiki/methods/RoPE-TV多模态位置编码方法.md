# RoPE-TV多模态位置编码方法

## 适用问题

在处理包含文本、图像或视频的多模态输入时，如何将多维的视觉空间坐标合理地编码至原有的 1D RoPE 中。解决不同模态之间的位置尺度不一致以及多模态内部相对距离失真的问题，避免直接展平所带来的信息损失或局部突变。

## 核心变换

将文本原本的一维位置拓展为二维或三维（如图像时的 $(n,n)$ 或视频时的 $(n,n,n)$），并将视觉信号的位置参数化为包含基础偏移 $\beta_i$ 与步长 $\gamma_i$ 的多维等差数列。依据以下三项等式（原则）求解：
1. **兼容性**：文本退化时能够兼容一维RoPE。
2. **等价性**：一段 $w \times h$ 大小的图像Patch等同于同等数量（$wh$）个文本Token在长序列中的长度消耗。
3. **对称性**：图像前后分别拼接相同数量的文本时，它们对图像的坐标增量影响对称。

## 典型步骤

1. 确立多模态在旋转位置编码上的维度一致性（例如图像采用2D坐标，视频采用3D坐标）。
2. 根据输入图像的宽高 $(w,h)$ 或者视频帧数 $(w,h,t)$，以及前序文本长度 $L$，列出对应维度的端点坐标。
3. 对齐前后文本的位置坐标关系，通过等价性与对称性反解出各维度上的初始偏移量（例如 $\beta_1 = L + \frac{1}{2}(wh - h)$ 等）。
4. 对不同模态使用各自确定的偏移与固定步长计算出的多维坐标，代入 RoPE 进行编码，并在后续 Attention 中实现模态融合。

## 直觉

如果图像的高宽不同，直接在对应维度分配长度会导致原本同样距离的两个 Patch 在编码空间的绝对差值变异。通过“统一将步长设为 1 并分配中心偏移量”，视觉模态内部可以保持尺度一致的稳定栅格结构，同时自然地使得视觉模态边界与文本之间产生较大的“位置跳跃”，从而起到类似于插入特殊标记（Special Token）的“模态隔离”效果。

## 边界

- **依赖预定义大小**：在自回归生成视频（三维）或不定长扩展时，如果偏移量依赖总时长（如 $t$），会导致必须提前知道生成多少帧视频，难以实现如纯文本那样无限制的无限自回归续写。
- **维度退化灾难**：如果不同维度的尺度跳跃设置不当，在极端长宽比下可能会使得编码密度过高或由于跳跃过大导致局部注意力断裂。

## 例子

对于 $w \times h$ 个 Patch 的图像，前序文本长度为 $L$。按照 RoPE-TV（RoPE-Tie-v2）：
文本的位置坐标为 $(n,n)$。
图像左上角第一个 Patch 坐标设为 $(\beta_1 + \gamma_1, \beta_2 + \gamma_2)$。
为满足上述三个原则且设 $\gamma_1 = \gamma_2 = 1$，求得偏移量 $\beta_1 = L + \frac{1}{2}(wh - h)$，$\beta_2 = L + \frac{1}{2}(wh - w)$。此时图像内相邻 Patch 的相对位置间隔严格固定为 $(0,1)$ 或 $(1,0)$，且图像到前后文本具有统一的较大跳跃间隔。
