# MathJax翻译与延时加载兼容方法

## 适用问题
适用于需要让网页数学公式兼顾自动整页翻译和动态延时加载的前端设计。否则，网页自带的翻译插件（如谷歌翻译）会把渲染前后或延时加载渲染过程中的 LaTeX 公式误翻译成乱码。

## 核心变换
通过引入时序上的状态机变换：先用 `notranslate` 标记全域阻止外部环境动作 -> 内部重构完毕（MathJax排版完成）并建立局部免疫区（仅给公式标记 `notranslate`） -> 解除全域封锁允许外部接管翻译。

## 典型步骤
1. 初始化时，为包含公式的文本块(例如标题和摘要)添加`notranslate`类。
2. 配置MathJax进行排版，在排版完成的回调中，对新生成的公式节点`.MathJax`添加`notranslate`类。
3. 在同一回调中，将最初添加在文本块上的`notranslate`类移除以触发自动翻译功能。
4. 结合延时加载逻辑，在加载新内容时，每次调用MathJax排版时也应用此回调策略，保证翻译始终安全按需执行。

## 直觉
浏览器插件常常监听 DOM 变化以自动介入。通过在初始阶段就禁止它们操作块级元素，给 MathJax 充足的时间来处理公式。等内部完成结构保护后，再把文本块的禁止标志撕掉，诱导翻译引擎开始工作，这样就能完美错开两者发力的时机。

## 边界
要求网页元素具有明确的选择器能锁定公式块。某些浏览器插件可能无视 `notranslate` 类，因此该方案主要针对遵循标准协议的翻译组件。

## 例子
在论文阅读网站中，使用此机制可以使得加载一篇新论文的摘要后，翻译立刻正确发生，且其中的数学公式不受损坏。
