## 适用问题

多轮对话生成中，对话历史由多轮交替发言组成，轮次可变、长度可变。传统 Seq2Seq 模型通常假设输入和输出是固定角色的两段文本，无法灵活建模多轮交替发言的模式。基于 LM 的多轮对话将对话历史拼接为单序列，使用单向语言模型统一建模，实现灵活的多轮生成。

## 核心变换

**输入**：多轮对话历史 $[u_1, r_1, u_2, r_2, \dots]$
**输出**：下一轮回复的生成概率分布
**核心**：将对话历史视为一段连续的文本序列

对话历史拼接为单序列：
$$
[CLS] \ u_1 \ [SEP] \ r_1 \ [SEP] \ u_2 \ [SEP] \ r_2 \ [SEP]
$$

使用单向语言模型建模：
$$
p(x_t | x_{<t}) = \text{softmax}(W h_t)
$$

说话者通过 Segment ID 区分：
- 说话者 A 的所有轮次使用 Segment ID = 0
- 说话者 B 的所有轮次使用 Segment ID = 1

## 典型步骤

1. **序列拼接**：将多轮对话历史用 [SEP] 连接成单序列
2. **Segment ID 分配**：不同说话者分配不同 segment id，使模型学会区分角色
3. **位置编码**：使用相对位置编码（如 NEZHA 的 RoPE）突破绝对位置限制
4. **单向 Mask 训练**：使用下三角 Mask 训练语言模型
5. **自回归解码**：在推理时逐步生成回复 token

## 直觉

核心思想：**对话即文本**。

多轮对话本质上是一段由不同角色交替发言的文本。既然单向语言模型可以生成连贯的文本段落，它同样可以生成连贯的对话——只要我们将对话历史按照时间顺序拼接起来。

关键设计是：
- **[SEP] 作为轮次分隔符**：类似于段落分隔符，帮助模型理解话语边界
- **Segment ID 区分角色**：让模型学会谁在什么时候说了什么
- **相对位置编码**：对话轮次可无限扩展，绝对位置编码存在上限

这种方法的优势在于极简：不需要编码器-解码器架构，不需要额外的对话状态追踪模块，一个预训练好的语言模型就能做多轮对话。

## 边界

- 拼接序列的总长度受限于模型的最大长度限制（通常 512 或 1024）
- 单向 LM 无法同时利用双向上下文信息，但对话生成本身是因果过程（只能从前往后生成），这反而是合适的
- 相比 Seq2Seq，缺乏编码器对输入的整体理解——模型只能从左到右逐步阅读对话历史
- 对长对话历史的建模能力弱于显式的对话状态追踪方法

## 例子

- DialoGPT：基于 GPT-2 的多轮对话模型，使用对话历史拼接训练
- NEZHA + 相对位置编码：突破绝对位置限制，可处理更长的对话历史
- 开放域闲聊：多轮交替发言的连贯性和角色一致性表现良好
