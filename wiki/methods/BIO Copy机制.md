# BIO Copy机制

## 适用问题
在诸如司法文书等高度严谨的长文本生成（如摘要生成）任务中，普通的 Pointer Network 难以保证生成的连续片段（n-gram）与原文本完全一致，容易引发专业词汇和实体错漏等致命错误。如何设计一种机制确保连续字词的忠实复制？

## 核心变换
在常规的词汇生成分布上，结合辅助任务增加一个并行的 BIO 标签预测，并在推理时用这些预测的标签对词汇输出分布（Logits）施加 Mask。
$$ p(y_t, z_t|y_{< t}, x) = p(y_t|y_{< t}, x) p(z_t|y_{< t}, x) $$
其中 $z_t \in \{\text{B}, \text{I}, \text{O}\}$。

## 典型步骤
1. 计算目标摘要与源文本的“最长公共子序列”。
2. 对于出现在最长公共子序列中的目标 token，根据它是否为单字或连续片段的一部分，打上 `B`（复制起点）或 `I`（连续复制）的标签。其他生成 token 打上 `O`（非复制）。
3. 在 Seq2Seq 模型的 Decoder 端，除了预测单词的概率分布外，新增一个序列预测任务，预测每个位置的 BIO 标签：$p(z_t|y_{<t}, x)$。
4. 推理阶段：先预测当前位置的 BIO 标签 $z_t$。
5. 如果 $z_t$ 是 `B`，在预测单词的词表中，将不在原文本中的词 mask 掉。
6. 如果 $z_t$ 是 `I`，在预测单词的词表中，将不能与前一个输出词构成原文中对应的 n-gram 的词 mask 掉。
7. 如果 $z_t$ 是 `O`，则正常自由生成。

## 直觉
由于 Pointer Network 每次只是独立计算每一个 token 的 Attention 分数并复制，它没有“片段”级别的整体意识，可能会把原文中不相邻的字拼凑在一起。而借鉴信息抽取任务中的 BIO 序列标注，我们可以让模型学会哪些词是连续的块。通过在解码时先预测 BIO，再用 BIO 反向约束输出词表，相当于引入了一套明确的语法约束机制，既保持了模型即插即用的灵活性，又严格杜绝了胡编乱造的 n-gram。

## 边界
BIO Copy 需要依赖精确的源-目标序列“最长公共子序列”对齐来生成监督信号。如果目标文本被大量改写而无法与源文本对齐，BIO 标签可能无法获取充足的监督信息。此外，在解码阶段预测标签错误可能会导致连续 mask 被截断，或者导致模型卡死在无候选词的困境中。

## 例子
“我 真的 非常 热爱 我 的 祖国”与“我 爱 我 的 祖国”对齐。生成“我(B) 爱(O) 我(B) 的(I) 祖国(I)”。在生成第二个“我”时预测出 B，限制输出在原文中挑选；下一步预测出 I，发现前面的词是“我”，在原文中查找“我”后面的词是“的”或者“真的”，从而限制下一步必定从这几个有效选项中抽取。

## 证据
- 8046 提到，法研杯司法摘要赛道中，BIO Copy 机制被首创使用。通过额外预测 B、I、O 并在预测时施加 mask，成功保证了片段的复制连续性，避免了错词，并且如果将其在 Encoder 和 Decoder 处同时施加还能提升同步性和效果。
