## 适用问题

适用于大规模 MoE 混合专家大模型训练中，由于缺少上下文导致前几层专家严重负载失衡的场景。

## 核心变换

在 MoE 的 Embedding 层后的前几层浅层网络中，根据预训练词表的 Token 词频分布，使用贪心算法生成静态映射表 tid2eid，替代复杂的 MoE 动态分配路由网络以实现负载均衡。

## 典型步骤

1. 统计并按降序排列模型预训练词表中所有 Token 的出现频率 $p_i$。
2. 设定 Expert 总数 $n$ 与每个 Token 需要激活的 Expert 数 $k$，并初始化各 Expert 负载向量 $f$ 为0。
3. 按频率从高到低逐个处理 Token，每次选出当前负载 $f$ 最轻的 $k$ 个 Expert。
4. 将这 $k$ 个 Expert 的 ID 记录到静态映射表 `tid2eid` 中。
5. 将当前 Token 的频率 $p_i/k$ 累加到选中的 $k$ 个 Expert 负载上，循环直到分配完毕。

## 直觉

前几层 MoE 没多少上下文信息，基于 Token 本身去选专家和预先写死每个 Token 去哪个专家其实差不多。既然可以写死，那如何保证每个专家分到的词语总频率差不多呢？这就是个装箱问题。我们从最常见的词开始，哪几个专家目前最闲，就把它丢给哪几个专家。这样贪心地分下去，最终所有专家的负载就会很均匀。

## 边界

如果某个 Token 的出现频率极端偏高（例如 $p_1 \gg k/n$），单个 Token 的分配就会超出理论均衡水平，导致无法实现严格的负载均衡。此时需要引入包含更多上下文的 $N$-gram Hash Routing。

## 例子

对词表大小 80000、Expert数 128、选 4 个 Expert 的网络，按此贪心法提前生成好 `tid2eid` 表。推理和训练时，Token $i$ 输入后，直接查表得到它对应的 4 个 Expert，免去了路由网络的计算开销和不均衡惩罚。

## 证据

- ev::11750::文章针对 DeepSeek V4 中的 first_k_hash 策略，推导了通过词频降序贪心算法生成静态分配表（tid2eid）来实现前期 MoE 负载均衡的方法。
