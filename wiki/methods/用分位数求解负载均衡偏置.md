# 用分位数求解负载均衡偏置

## 适用问题

大型语言模型训练中的 MoE（混合专家）架构经常遭遇“马太效应”：Router 总是更偏向调用那几个“明星” Expert，导致大多数 Expert 闲置甚至能力退化，导致计算集群负载严重失衡。常规通过增加 Auxiliary Loss 的方法超参极难调节，调高了损害语言建模性能，调低了又没用。我们需要一种无需调参、即插即用的刚性物理级均衡手段。

## 核心变换

$$ b_j = \text{Quantile}(\boldsymbol{s}_{\cdot, j}, \ 1 - k/n) \implies \text{active if } s_{i,j} > b_j $$
将从 Loss 中迭代出的“动态偏置”，转化为沿着序列批次维度的“特征分数分布统计”，把求解一个最优化问题用一个极度轻量的分位数提取操作闭式解代替。

## 典型步骤

1. **解耦分配与打分**：首先承认并保持原有的 Router 打分网络（$\boldsymbol{s}$）不进行惩罚，使其仅关心语言建模；而在决定哪些 Expert 上场时，对每个专家的分数增加一项外部的辅助偏置向量 $\boldsymbol{\beta}$。
2. **重塑选择机制（Expert Choice视角）**：转变观念，不是每个词去找它最爱的几个专家，而是如果一共有 $m$ 个样本词、指定每个词分配 $k$ 个专家、总共 $n$ 个专家，那么为了绝对负载均衡，每个专家就必须在全批次中挑选最想来它这儿的前 $mk/n$ 个词。
3. **转化为分位数判定**：专家“挑词”的阈值底线，在数学上等价于要求该专家在当前 Batch 中的全部打分里，取出第 $mk/n + 1$ 大的那个分数值。这刚好对应着该数值分布的“$1-k/n$ 分位数（Quantile）”。
4. **单步截断应用**：对于每个批次计算该分位数作为 $\boldsymbol{\beta}$ 后，当词对某专家的打分减去 $\boldsymbol{\beta}$ 大于零时即予以激活，直接在数学上保证了这批数据能被均摊地派发，省去了额外的反向传播迭代。

## 直觉

假设有5个饭堂打饭大妈（专家）和100个学生（Token），以往学生挑饭堂，总是全挤到其中一个打菜最多的阿姨那里，导致她忙死、别人闲死。给大妈加辅助 Loss 就像校长天天扣这阿姨工资逼她赶走学生，很伤感情。分位数法则像是直接让阿姨来挑学生：既然必须绝对均摊，每个阿姨定额只能打20份饭。那就直接拉出想来该阿姨这儿排队的学生分数表，按对她感情深厚程度排序，恰好卡到第20名学生的那个分数底线（分位数偏置）。只要超过这个底线，你就进，不到底线你去别家。这就完美在学生喜爱度与绝对公平接客中找到了平衡点。

## 边界

- 分位数在包含极大规模全局样本时（比如极长序列并行或大 Batch Size 分布在数百张卡上），做一次全局 Sort/Quantile 会造成昂贵的通信瓶颈。在实践中只能在局部微批次内求取分位数并用平滑移动平均。
- 训练初期由于初始化重叠度高，如果采用同步的分位数可能引发全员匹配到同一个阈值。因此在第一步必须配合合适的参数正态初始假设来引入预设 $\beta_0$ 防止信息泄漏或爆炸。

## 例子

在自建的模型库训练中，使用 DeepSeek 首创的 Loss-Free MoE。我们不使用 `SignSGD` 花费数百步在辅助网络里调优 Bias，而是直接调包 `torch.quantile` 沿着 Batch 大小去切分布的百分比点位。替换这行代码后，网络分配绝对均衡且完全没有损伤主任务交叉熵 Loss。
