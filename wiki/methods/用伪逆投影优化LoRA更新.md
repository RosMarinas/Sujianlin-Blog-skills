# 用伪逆投影优化LoRA更新

## 适用问题

标准的 LoRA 将权重更新限制在低秩子空间内，导致其优化轨迹和能力上限通常不及全参数微调。如何在不增加推理负担和可训练参数量的前提下，从优化器层面让 LoRA 每一步的更新效果尽量贴合甚至等价于全量微调的一步？

## 核心变换

$$ \mathop{\text{argmin}}_{H_A, H_B} \left\| H_A B + A H_B - G \right\|_F^2 \implies \begin{cases} H_A = G_A (B B^\top)^{-1} + AC \\ H_B = (A^\top A)^{-1} G_B (I - B^\top(B B^\top)^{-1} B) - CB \end{cases} $$
将传统基于标量学习率标定梯度方向的 SGD 更新，转变为通过矩阵伪逆对全量梯度进行子空间正交投影，解出低秩参数上的理论最优步进量。

## 典型步骤

1. **获取全量梯度**：对于原权重矩阵，正常反向传播求出其若完全自由更新时所获得的全局理想梯度 $G$。
2. **构建目标函数**：由于 LoRA 在当前步的实际作用是将权重变为 $W_t + H_A B + A H_B$，我们将此更新量与 $G$ 做差，目标是将其 Frobenius 范数最小化。
3. **求偏导找极值**：利用矩阵求导法则和伪逆的性质，分别固定 $H_B$ 求 $H_A$，固定 $H_A$ 求 $H_B$。
4. **施加不变性化简**：由于 $A(CB) - (AC)B = 0$ 存在内在自由度，解析解可以被化简为只依赖局部梯度 $G_A = G B^\top$ 和 $G_B = A^\top G$ 以及 $A, B$ 的内部协方差的表达式形式。
5. **执行更新**：根据求出的最优 $H_A$ 和 $H_B$ 直接更新 $A$ 和 $B$，使这一步在数学上达到当前低秩维度下最贴近全量更新的效果。

## 直觉

普通的 LoRA 就像是让你只能拉动两根线来控制木偶的动作。如果随便凭感觉拉，动作肯定不如十指全上（全量微调）来得准确。但是伪逆投影算法就是帮你瞬间算出一个数学上“在两根线限制下最像十指全上动作的力度组合”。用伪逆去修正，就保证了只要是在这个维度的能力范围内，它走的就是最完美的一条捷径。

## 边界

- 矩阵求逆或伪逆在深度学习中往往存在数值稳定性问题；当 $A$ 或 $B$ 的某些行或列由于初始化原因极小或相关度极高时，求解 $(BB^\top)^{-1}$ 时需要引入截断或极小正常数（即正则化）。
- 这种做法显著增加了每次反向传播计算梯度时的显存消耗与矩阵计算量，属于“以计算量换收敛性”的方法。

## 例子

LoRA-Pro 在计算每一步更新时，不仅计算关于 $A, B$ 的朴素导数，还在此导数上右乘 $(BB^\top)^{-1}$ 以及做相应的投影消除，严格对齐了第一步到最后一步的模型更新轨迹，在各种微调任务上比普通 LoRA 收敛更快，且缩小了与全量微调的差距。
