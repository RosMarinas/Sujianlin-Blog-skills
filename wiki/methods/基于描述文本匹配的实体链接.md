# 基于描述文本匹配的实体链接

## 适用问题

适用于实体有较丰富描述字段、同名候选需要通过上下文和描述文本消歧的实体链接任务。

## 核心变换

结构化候选实体 -> 连续描述文本 -> query-描述文本匹配分类。

## 典型步骤

1. 为同名候选实体整理名称、义项和属性描述。
2. 分别编码 query 文本和候选实体描述文本。
3. 在 query 中标记待链接实体 span。
4. 融合 query/span 与描述编码，逐候选二分类并取最高概率实体。

## 直觉

实体链接的消歧信息常藏在属性和义项里；把它们改写为文本后，可复用文本匹配模型比较上下文和候选描述。

## 边界

逐同名候选遍历会带来较低采样效率和更长训练/预测时间；证据不支持全库一次性召回。

## 例子

- 6919 明确说每个实体对应一段较长文本描述，实体链接可转成 query 文本、实体 span 与实体描述的匹配二分类。
