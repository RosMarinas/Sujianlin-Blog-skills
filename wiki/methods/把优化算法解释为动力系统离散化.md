# 把优化算法解释为动力系统离散化

## 适用问题

当一个训练现象主要由优化器迭代方式决定，而不是由模型结构或数据语义决定时，可以尝试把离散更新改写为连续动力系统。典型触发问题包括：GD 为什么会收敛到极小值点、SGD 的 batch size 和学习率为什么影响探索范围、Momentum 为什么可被看成加速、以及某些训练过程为什么会或不会退化。

## 核心变换

这个方法的输入是离散优化更新式，输出是 ODE 或 SDE 形式的连续动力系统。以 `5655` 为基本例子，GD 更新先被改写成差分商

```tex
\frac{\boldsymbol{\theta}_{n+1}-\boldsymbol{\theta}_{n}}{\gamma}
=-\nabla_{\boldsymbol{\theta}}L(\boldsymbol{\theta}_n)
```

然后在小学习率下把左侧视为时间导数，得到梯度流 ODE。SGD 的随机子集梯度进一步被写成全量梯度加估计误差，误差在源文中近似为高斯噪声，于是 ODE 变成带噪声的 SDE。

## 典型步骤

1. 从源文中的离散更新式出发，不先抽象成口号。
2. 把更新式除以步长，形成差分商。
3. 在小步长假设下，把差分商解释为连续时间导数，得到 ODE。
4. 如果更新使用随机子集或随机扰动，把估计误差显式写成噪声项，得到 SDE。
5. 分析连续系统的轨迹、不动点、平衡分布或摩擦项。
6. 将连续分析结果翻译回原优化器设计，例如 batch size、学习率衰减或 Momentum 的解释。

## 直觉

离散优化器每一步看起来只是参数更新，但当步长足够小时，它可以被看成沿某个连续系统轨迹前进。这样一来，学习率、噪声、动量和稳定性不再只是调参经验，而能分别对应步长、扩散噪声、二阶系统摩擦项或数值误差控制。

## 边界

- 小步长近似是核心前提；学习率过大时，连续极限解释只能作为启发。
- SGD 到 SDE 依赖对随机梯度误差的近似建模，源文明确说这是半定性、半定量分析。
- 该方法解释的是优化器行为，不应直接拿来解释由模型结构、数据分布或目标函数语义主导的问题。
- 跨系列迁移时，只能迁移“离散过程到连续动力学”的问题结构，不能把某篇文章的具体结论直接搬到新场景。
