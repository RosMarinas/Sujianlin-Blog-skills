# Positional Interpolation (PI)

## 适用问题

大型语言模型在遇到超出其预训练长度 $N$ 的上下文时，如果直接输入大于 $N$ 的位置编码（直接外推），效果会急剧下降，需要一种机制来支持数倍长度的长文本理解。

## 核心变换

从空间外推转向空间压缩：将超出训练范围的超大位置编号等比例缩小，插值回模型见过的有效位置区间（$[0, N]$ 内）。

## 典型步骤

1. **确定缩放因子**：假定训练时最大长度为 $N$，目标推理长度为 $k \times N$，则设定缩放系数为 $k$。
2. **位置压缩**：将待查询的位置坐标 $n$ 除以 $k$，变为 $n/k$。
3. **坐标映射**：使用 $n/k$ 输入至原有的位置编码公式（如 RoPE），由此计算出的位置特征仍完全处于 $[0, N]$ 的定义域内。
4. **少量微调**：由于将 $n$ 与 $n+1$ 的间隔变成了 $1/k$，相邻 Token 的相对高频距离被打破，必须使用长文本进行小步长微调以恢复对距离密集度的适应。

## 直觉

想象你有一个只标了 0 到 100 刻度的温度计，现在你要测量 200 度。外推法是试图在玻璃管上面凭空画新的刻度，模型根本不认识；内插法就是把刻度的物理距离缩小一倍（变成 0.5 步进），使得 200 度对应到物理的 100 刻度线上。虽然每个格子代表的间隔变挤了，但模型只要稍微学习一下就能重新适应。

## 边界

- **破坏相对距离度量**：强行将相邻位置距离从 1 压缩到 $1/k$，干扰了高频维度捕捉相邻 Token 精细相对顺序的能力。
- **强制微调需求**：不同于 NTK-aware Scaled RoPE 能做到零样本外推，单纯的线性位置内插由于打破了局部序列关系，通常必定需要微调模型。

## 例子

- **Meta 的 LLM 上下文扩展实验**：对于采用 RoPE 的语言模型，通过将位置坐标统一除以目标扩展倍数，再加上很少步数的微调，成功使模型能够理解远超预训练长度的 Context。
