# 用重定向扩展实现曲线救国搜索

## 适用问题

自动网络请求拦截、绕过网络访问限制、学术文献镜像快捷跳转。当自建学术文献阅读平台（如 Cool Papers）希望为用户提供便捷访问入口，但又无法投入大量资源开发独立搜索引擎时，可以通过浏览器扩展的请求重定向机制，将用户在其他平台的搜索行为”导向”自建平台。

## 核心变换

将不可达或低效的网络请求 URL 在浏览器底层（扩展程序的 background 层）动态重写为可用镜像站或定制服务的 URL。其本质是一种”借花献佛”式的搜索方案——利用成熟的第三方搜索引擎发现文献标识符，再利用重定向扩展将标识符映射到自建平台。

## 典型步骤

1. 编写浏览器扩展 background 脚本监听 webRequest 接口
2. 正则匹配不可达的搜索链接或文献请求 URL（如 arXiv、OpenReview、ACL 等论文源的链接）
3. 替换目标域名为镜像站或公开可达地址并返回重定向响应
4. 可选：在论文详情页注入快捷跳转按钮，支持一键直达

## 直觉

当我们无法直接在服务端构建一个庞大的全局搜索引擎时，可以”借花献佛”，利用浏览器扩展的重定向能力，使用户用任意传统搜索引擎（如Google或Arxiv搜索）查找到目标，然后通过前端右键或拦截自动跳转到我们提供的定制化解析页面。

## 边界

仅能处理用户能够通过公开途径检索到标识符（如 Arxiv ID）的情况；依赖于浏览器扩展环境（如 Chrome 的 webRequest 或 declarativeNetRequest API），不适用于脱离浏览器的纯 API 交互；如果自建服务端存在速率限制，需设计多优先级队列防止服务端被压垮。

## 例子

- **基础重定向**：在 Google 搜索到论文后，右键点击”Redirect to Cool Papers”，将 `arxiv.org/abs/xxxx.xxxxx` 重定向为 `papers.cool/arxiv/xxxx.xxxxx`。
- **多论文源支持**：v0.2.0 版扩展支持 arXiv、OpenReview、ACL、IJCAI、PMLR 共五个论文源，并自动在页面中插入快捷跳转链接。
- **多 ID 处理**：新版扩展支持同时检测所选文字、超链接、页面路径中的多个论文 ID，并在同一窗口打开对应标签页。

## 证据

- ev::9978::文章介绍了通过编写 Chrome 重定向扩展，让用户在任意页面选中论文后右击即可跳转到 Cool Papers 网站，从而”借花献佛”地实现了全局搜索功能。
- ev::10480::文章记录了扩展升级至 v0.2.0 版本的改进，包括多论文源支持、PDF 页面跳转、站内划词搜索以及页面快捷链接的插入。
