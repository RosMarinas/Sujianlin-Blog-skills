## 适用问题

针对各类缺乏统一 HTML 模板的论坛回帖、博客评论等网页数据进行通用抓取时，需要将混成一坨的“主帖”与“不同层级的回复”无监督地自动切分切割。

## 核心变换

将“复杂的网页二维视觉/嵌套层级分块问题”转换为“一维标号序列的一阶差分峰值寻找问题”。构建一维位置编号这一辅助序列，以序列数值的跳跃来表征DOM树结构的逻辑跃迁。

## 典型步骤

1. **DFS 遍历标号**：通过深度优先搜索（DFS）遍历网页的 DOM 树，并为每个访问到的文本节点分配一个顺序位置编号。
2. **例外标号压缩**：在遍历过程中，对纯样式标签（如 strong, em, a 等）设置例外规则，在进入这些标签时不变更层级距离惩罚，以确保段落内由于样式被切碎的文本编号依然紧密。
3. **差分计算**：提取所有过滤后的有效文本内容，保留其对应的 DFS 顺序编号，形成一个离散位置序列。对该位置序列做一阶差分。
4. **峰值聚类**：差分序列中数值显著增大的局部最大值（峰值）代表了 DOM 树上的长距离回溯跳跃。这些跳跃峰值点即作为不同逻辑楼层（回帖块）的天然切割边界。

## 直觉

当你一目十行地在一个长论坛里阅读时，虽然不同的跟帖长短不一、内容各异，但你一眼就能看出哪里是一楼、哪里是二楼，因为每换一个楼层，网页结构都会有一个明显的“框架重置”。如果把眼睛阅读文本的顺序用DOM树的搜索路径标上数字：同一个回复里的几句话离得很近，标号可能就是 10, 11, 12；一旦跳到下一个回复，因为要跨越很多中间无关的包裹框，下一个有效文本的标号可能直接蹦到了 35。因此，只要我们对标号做一次减法（差分），看到谁突然暴涨形成一个“山峰”，那里毫无疑问就是上一楼和下一楼的分界线。

## 边界

- 依赖于 HTML DOM 树本身的物理结构能够体现逻辑层次结构。如果网页用 JS 乱序动态渲染，或刻意平铺了所有结构而单纯靠 CSS 绝对定位来制造视觉楼层，该方法会失效。
- 需要恰当设定峰值选取的平滑或阈值参数，如果一个帖子内部包含极其复杂的巨大表格，可能会被误认为是一个差分峰值导致单层断裂。

## 例子

爬取某论坛页面文本：
有效内容节点及其 DFS 标号：
- `[15]` "楼主帖子标题"
- `[18]` "今天天气真好，出去玩了。"
- `[32]` "一楼回复："
- `[34]` "羡慕楼主啊"
- `[50]` "二楼回复："
- `[51]` "求攻略！"

差分序列：
- `18 - 15 = 3`
- `32 - 18 = 14` (峰值！) -> 分界点，区分楼主和一楼。
- `34 - 32 = 2`
- `50 - 34 = 16` (峰值！) -> 分界点，区分一楼和二楼。
- `51 - 50 = 1`

通过峰值14和16，自动切分出三块文本集合。
