## 适用问题

基于大规模检索系统和 Seq2Seq 模型构建的抽取式或生成式问答系统（如基于网页搜索结果的机器阅读理解），面对同一问题拥有多个参考文档片段，需要整合多视角文档得出单一的最优短文本答案。

## 核心变换

将常规的“每个文档单独解码到底后，依据最终整句置信度打分投票”转换为“在词级别单步（Step-by-Step）的概率对数分布求均值融合”。将篇章投票的融合发生点从解码后拉入解码中。

## 典型步骤

1. **并行送入**：将用户提问分别与 $N$ 个召回的检索篇章拼接成 $N$ 个独立的输入，送入预训练 Seq2Seq 模型（如 UNILM）。
2. **过滤无答案片段**：在解码生成首个字符时，观察各篇章分支的概率预测。如果某篇章首字大概率预测为结束符 `[SEP]`，判定该片段无答案并将其从集合中直接剔除。
3. **单步概率融合**：对于剩余存活的篇章分支，获取它们在当前时刻对整个词表的对数概率输出（Logits）。将这些概率在 Token 维度上进行算术平均（或带权平均）。
4. **统一 Beam 裁剪**：基于融合后的单字联合概率，挑选概率最高的 Top-K 个字（Beam 大小）。
5. **状态同步步进**：将挑选出的 Top-K 个字作为已知前缀，反馈给所有存活的篇章状态，并行推演下一步分布并继续循环，直至全部触发结束符。

## 直觉

普通的投票法就像是让几个学生拿着不同的资料独立做卷子，做完后看谁底气足就交谁的卷子；但由于阅读理解的答案往往是碎片化的，单靠一个人的底气可能选出片面的答案。而这个方法则是让这几个学生“每写一个字之前先开个会”讨论一下下一个字写什么，把大家都认同度最高的那个字写下来，再根据写下的字继续讨论下一个字。这种单字步进的投票极大降低了产生长句幻觉的风险，让最终解码序列高度集中在各篇章共有的确凿信息上。

## 边界

- 运算复杂度高。原先的推理可以独立批处理结束，现在每一层循环都需要对概率合并、重新分配，并强行阻断了单文档在内部可能存在的独特长距连贯性。
- 如果多数检索文章都在“胡说八道”，这种平均分布会使得模型产生极大的熵增震荡，最终选出平庸且无意义的频繁字拼接。

## 例子

提问：“谁是宋朝第一个皇帝？”
检索出3篇文章。
模型在生成第一个字时：
- 篇章 A 的概率输出：赵 (90%)，[SEP] (5%)
- 篇章 B 的概率输出：[SEP] (99%) -> 判定为无答案，本轮直接放弃 B 篇章参与！
- 篇章 C 的概率输出：赵 (60%)，宋 (30%)
保留 A 和 C 进行平均，字库中“赵”的平均概率为 (90% + 60%) / 2 = 75%，远超其他字。
于是将“赵”选中，接着在给定的前缀“赵”下，篇章A和C继续预测下一个字，直到拼出完整的“赵匡胤”。

## 证据

- ev::7115::一种跟beam search更加“契合”的方案：先排除没有答案的篇章，然后在解码答案的每一个字时，直接将所有篇章预测的概率值（按照某种方式）取平均。
- ev::7115::具体来说，所有篇章分别和问题拼接起来，然后给出各自的第一个字的概率分布。那些第一个字就给出[SEP]的篇章意味着它是没有答案的，排除掉它们。
- ev::7115::排除掉之后，将剩下的篇章的第一个字的概率分布取平均，然后再保留topk（beam search的标准流程）。预测第二个字时，每个篇章与topk个候选值分别组合，预测各自的第二个字的概率分布，然后再按照篇章将概率平均后，再给出topk。
